# 从 V2 升级到 V3

V3 保留既有的 `data.json` 和自定义模板，不会自动删除用户数据。首次加载会把旧模板以 `legacy` 兼容主题接入 ThemeManifest 注册表。

## 升级前

1. 关闭 Obsidian，备份库中的 `.obsidian/plugins/yh-mp-preview/data.json`。
2. 用 BRAT 安装或更新到 V3；插件目录必须仍为 `yh-mp-preview`。
3. 更新后重新打开 Obsidian，并在“设置 → 第三方插件”确认插件已启用。

## 已知的兼容处理

- V2 中不完整的自定义模板不会阻止插件启动；缺失字段会使用安全的默认值。
- 已移出发行版的 xiaohu 主题不会被静默替换为第三方来源主题。若它曾被选中，V3 会切回 `default`，并保留原主题 ID 作为迁移记录。
- 文章配方在 V3 才真正参与预览和复制；“legacy-compatible”用于保持旧文稿的最小变化。

## 回退

如需回到 2.0.10，先备份 V3 的 `data.json`，再由 BRAT 安装旧标签。V3 新增的文章配方、布局快照和 ThemeManifest 元数据会被旧版忽略；自定义模板本身仍采用旧格式保存。

## 升级后检查

1. 打开任意 Markdown 文档，确认预览可见。
2. 选择一个主题和文章配方，点击“检查”。
3. 有错误时先修复，再复制到公众号编辑器。
4. 需要反复试版式时，用工具栏“保存布局”和“恢复最近布局”。
